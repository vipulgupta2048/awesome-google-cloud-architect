## üöÄ How to Contribute

### 1. Fork and Clone
```bash
# Fork the repository on GitHub, then:
git clone https://github.com/YOUR-USERNAME/gcp-professional-cloud-architect-notes.git
cd gcp-professional-cloud-architect-notes
```

### 2. Create a Feature Branch
```bash
git checkout -b feature/your-improvement-name
# Examples:
# git checkout -b fix/broken-links-vpc
# git checkout -b add/cloud-functions-examples
# git checkout -b update/iam-policies-2025
```

### 3. Make Your Changes
- Follow the existing structure and naming conventions
- Use Obsidian linking format: `[[Note Name]]`
- Add images to the `images/` folder
- Update index files when adding new content

### 4. Test with Obsidian
- Open the repository as an Obsidian vault
- Ensure all `[[internal links]]` work correctly
- Check that new content integrates well with existing notes

### 5. Commit and Push
```bash
git add .
git commit -m "Add: Clear description of your changes"
git push origin feature/your-improvement-name
```

### 6. Create Pull Request
- Use a clear, descriptive title
- Explain what you changed and why
- Reference any relevant issues

## üìã Contribution Guidelines

### File Structure
- Keep the existing folder organization
- Use descriptive file names with proper capitalization
- Follow the naming pattern: `G ServiceName.md` for Google services

### Content Standards
- **Accuracy**: Ensure all technical information is correct
- **Clarity**: Write for exam candidates, not experts
- **Currency**: Use up-to-date information (2024-2025)
- **Attribution**: Credit sources when using external content

### Obsidian Formatting
- Use `[[Double Brackets]]` for internal links
- Include `#tags` for categorization
- Add table of contents for long documents:
  ```markdown
  ```table-of-contents
  title: 
  style: nestedList
  minLevel: 0
  maxLevel: 0
  includeLinks: true
  ```
  ```

### Images and Media
- Place all images in the `images/` folder
- Use descriptive filenames: `gke-cluster-architecture.png`
- Optimize image sizes (under 1MB when possible)
- Include alt text: `![GKE Architecture](images/gke-cluster-architecture.png)`

## üîç Content Review Process

### Before Submitting
- [ ] Content is technically accurate
- [ ] Links work correctly in Obsidian
- [ ] Images are properly referenced
- [ ] Spelling and grammar are correct
- [ ] Follows existing formatting style

### Review Criteria
1. **Technical Accuracy**: Information must be current and correct
2. **Exam Relevance**: Content should be useful for PCA certification
3. **Integration**: New content should connect well with existing notes
4. **Quality**: Clear writing, proper formatting, good organization

## üö´ What Not to Contribute

- **Copyrighted Material**: Don't copy from official Google documentation without proper attribution
- **Exam Dumps**: No actual exam questions or answers
- **Personal Information**: Remove any personal accounts, project IDs, etc.
- **Promotional Content**: No advertising or self-promotion
- **Off-Topic Content**: Keep focus on GCP Professional Cloud Architect topics
